## React.js原理

> react 新的编程思想、编程方式。React JS 可以用react的语法写网页的交互效果，React Native 可以用react语法编写原生的APP应用。
>
> React JS 是由Facebook在2013年5月推出的一款前端框架，带来一种全新的函数式的编程风格
>
> 16version之后的版本称为 React Fiber，因为16版本中react底层在事件循环中加入了优先级等一些概念，可以利用事件循环的一些碎片时间执行一些高优先级的用户交互，提高reactjs使用过程中的用户体验。
>
> react相对vue更灵活些，处理更复杂的业务会有更多的选择；vue提供了更丰富的API，功能更简单，正因为API多，所以灵活性受到了限制。复杂度比较高的项目更倾向react，面向用户复杂度不高的选用vue。

**解决**

1. 传统UI操作关注太多细节
2. 应用程序状态分散在各处，难以追踪和维护
3. 传统MVC难以扩展和维护

React：始终整体刷新页面

Flux：单向数据流

```react
// App.js
import React, { Component } from 'react'

class App extends Component {
    constructor(props) {
        super(props)
        // state数据改动需要通过this.setState()
        this.state = {
            inputValue: '',
            list: []
        }
        // 改变this指向
        this.handleInputChange = this.handleInputChange(this)
    }
    render() {
        // jsx语法，通过react底层支持
        return (
            {/*JSX注释*/}
            {
            	// 注释
            }
            <React.Fragment>
                <div>
                	<label htmlFor="insertArea">输入内容</label>
                    <input 
						id="insertArea"
						className="input"
            			value={this.state.inputValue}
    					onChange={this.handleInputChange}/>
					<button></button>
                </div>
            	<ul>
                    {
                        this.state.list.map((item, index) => {
                            return (
                               <li 
                                	key={index} 
                                	onClick={this.handleItemDelete.bind(this, index)}
                                    {/*不转义HTM*/}
                                    dangerouslySetInnerHTML={{__HTML: item}}
								>
                                	{item}
								</li>
							)
                          })
                        }
                </ul>
            </React.Fragment>
        )
    }
    handleInputChange(e) {
        const value = e.target.value
        // 推荐用法,异步(prevState为改变前的state)
        this.setState((prevState) => {
            return {
                inputValue: value
            }
        })
        this.setState({
            inputValue: e.target.value
        })
    }
	handleBtnClick() {
        this.setState({
            this.setState({
				list: [...this.state.list, this.state.inputValue]
            }),
            inputValue: ''
        })
    }
    handleItemDelete(index) {
        const list = [...this.state.list]
        list.splice(index, 1)
        this.setState({
            // es6 list:list 简写
            list
        })
    }
}
export default App
```

### JSX

> 在react中在在js文件里写的HTML标签称为jsx语法
>
> 可以使用自己的组件作为标签名，首字母必须大写
>
>  JSX，是一个 JavaScript 的语法扩展，可以很好地描述 UI 应该呈现出它应有交互的本质形式
>
> 在编译之后，JSX 表达式会被转为普通 JavaScript 函数调用，并且对其取值后得到 JavaScript 对象

1. JSX本身也是表达式

   ```react
   const element = <h1>Hello word!</h1>
   ```

2. 在属性中使用表达式

   ```react
   <MyComponent foo={1 + 2 + 3} />
   ```

3. 延展属性

   ```react
   const props = {first: 'Ben', lastName: 'Hector'}
   const greeting = <Greeting {...props} />
   ```

4. 表达式作为子元素

   ```react
   const element = <li>{props.message}</li>
   ```

**约定**

1. 自定义组件以大写字母开头
   - React认为小写的tag是原生DOM节点，如div
   - 大写字母开头的为自定义组件
   - JSX标记可以直接使用属性语法，例如<menu.item/>

### 虚拟DOM

```markdown
若无虚拟DOM
1. state数据
2. JSX模板
3. 数据 + 模板 结合。生成真实的DOM，来显示
4. state 发生改变
5. 数据 + 模板 结合，生成真实的DOM，替换原始的DOM
缺陷：
第一次生成了一个完整的DOM片段
第二次生成了一个完整的DOM片段
第二次的DOM替换第一次的DOM，非常耗性能
```

```markdown
若无虚拟DOM
1. state 数据
2. JSX 模板
3. 数据 + 模板 结合，生成真实的DOM，来显示
4. state 发生改变
5. 数据 + 模板 结合，生成真实的DOM，并不直接替换原始的DOM
6. 新的DOM（DoucumentFragment)和原始的DOM做对比，找差异
7. 找出input框发生了变化
8. 只用新的DOM中的input元素，替换掉老的DOM中的input元素
缺陷：
性能提升不明显
```

```markdown
虚拟DOM
1. state 数据
2. JSX 模板
   JSX->React.createElement->虚拟DOM（JS对象）->真实的DOM
4. 数据 + 模板 结合，生成虚拟DOM（虚拟DOM就是一个JS对象，用它来描述真实DOM）（损耗了性能）
   ['div',{id:'abc'},['span',{},'hello']]
3. 用虚拟DOM的结构生成真实的DOM，来显示
   <div id="abc"><span>hello</span></div>
5. state 发生变化
6. 数据+ 模板 生成新的虚拟DOM（极大地提升了性能）
   ['div',{id:'abc'},['span',{},'world']]
7. 比较原始虚拟DOM和新的虚拟DOM的区别，找到区别是span中内容（极大地提升了性能）
8. 直接操作DOM，改变span中的内容
优点：
1. 用js对象的创建和对比替换掉了DOM的创建和对比，实现了极大地性能飞跃
2. 它使得跨端应用得以实现。React Native
```

 **Diff 算法**

两个虚拟DOM的比对

1. 同级比较
2. 同级比较时不同，该节点下DOM全部删除，重新生成DOM更新（算法简单，比对算法快）



### 生命周期

生命周期函数指在某一时刻组件会自动调用执行的函数

![image-20201105201120283](/img/image-20201105201120283.png)

**componentWillMount**

在组件即将被挂载到页面的时刻，自动被执行

**render**

render必须存在，其他生命周期函数在React.Component中内置了

**componentDidMount**

组件被挂载到页面之后，自动被执行

**shouldComponentUpdate**（需返回Boolean类型的结果）

```react
shouldComponentUpdate(nextProps, nextState) {
    
}
```



组件被更新之前，自动被执行

**componentWillUpdate**

组件被更新之前，自动被执行，但它在shouldComponentUpdate之后被执行，如果shouldComponentUpdate返回true才执行

**componentDidUpdate**

组件被更新完成之后，自动被执行

**componentWillReceiveProps**

子组件从父组件接收了参数，父组件的render函数被执行了，子组件的这个生命周期函数就会被执行

**componentWillUnmount**

当这个组件即将被从页面中剔除的时候，自动被执行

### 使用 Prop Types 类型检查

```react
class Greeting extends React.Component {
  render() {
    return (
      <h1>Hello, {this.props.name}</h1>
    );
  }
}

Greeting.propTypes = {
  // string/number/bool/symbol/object/array/func
  name: PropTypes.string,
  // 任何可被渲染的元素（包括数字、字符串、元素或数组）
  optionalNode: PropTypes.node,
  // 一个 React 元素
  optionalElement: PropTypes.element,
  // 一个 React 元素类型（即，MyComponent）
  optionalElementType: PropTypes.elementType,
  // 你也可以声明 prop 为类的实例，这里使用JS 的 instanceof 操作符
  optionalMessage: PropTypes.instanceOf(Message),
  // 你可以让你的 prop 只能是特定的值，指定它为枚举类型
  optionalEnum: PropTypes.oneOf(['News', 'Photos']),
  // 一个对象可以是几种类型中的任意一个类型
  optionalUnion: PropTypes.oneOfType([
    PropTypes.string,
    PropTypes.number,
    PropTypes.instanceOf(Message)
  ]),
  // 可以指定一个数组由某一类型的元素组成
  optionalArrayOf: PropTypes.arrayOf(PropTypes.number),
  // 可以指定一个对象由某一类型的值组成
  optionalObjectOf: PropTypes.objectOf(PropTypes.number),
  // 可以指定一个对象由特定的类型值组成
  optionalObjectWithShape: PropTypes.shape({
    color: PropTypes.string,
    fontSize: PropTypes.number
  }),
  // 你可以在任何 PropTypes 属性后面加上 `isRequired` ，确保这个 prop 没有被提供时，会打印警告信息
  requiredFunc: PropTypes.func.isRequired,
  // 任意类型的数据
  requiredAny: PropTypes.any.isRequired,
};

// 指定 props 的默认值：
Greeting.defaultProps = {
  name: 'Stranger'
};
```
