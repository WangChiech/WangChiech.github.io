## Node.js原理

> react 新的编程思想、编程方式。React JS 可以用react的语法写网页的交互效果，React Native 可以用react语法编写原生的APP应用。
>
> React JS 是由Facebook在2013年5月推出的一款前端框架，带来一种全新的函数式的编程风格
>
> 16version之后的版本称为 React Fiber，因为16版本中react底层在事件循环中加入了优先级等一些概念，可以利用事件循环的一些碎片时间执行一些高优先级的用户交互，提高reactjs使用过程中的用户体验。
>
> react相对vue更灵活些，处理更复杂的业务会有更多的选择；vue提供了更丰富的API，功能更简单，正因为API多，所以灵活性受到了限制。复杂度比较高的项目更倾向react，面向用户复杂度不高的选用vue。

### 脚手架：create-react-app

```bash
npm install -g create-react-app
create-react-app 项目名
```

**项目中引用`registerServiceWorker`**

```js
// PWA progressive web application，理念是通过写网页的形式写一些手机的APP应用，引用registerServiceWorker实际上就是帮助我们借助网页写手机APP应用的一个功能。可以做到的效果是，网页应用上传到支持HTTPS协议的服务器上，当第一次访问时需要联网才能访问我们的应用，之后一旦断网，如果引用了registerServiceWorker，第二次依然可以看到之前访问的页面，registerServiceWorker帮助把之前访问的网页存储到浏览器之内
import registerServiceWorker from './registerServiceWorker'
```

**`manifest.json`文件**

如果网页当成APP应用使用，在该文件定义快捷方式、图标等

**JSX**

> 在react中在在js文件里写的HTML标签称为jsx语法
>
> 可以使用自己的组件作为标签名，首字母必须大写
>
>  JSX，是一个 JavaScript 的语法扩展，可以很好地描述 UI 应该呈现出它应有交互的本质形式
>
> 在编译之后，JSX 表达式会被转为普通 JavaScript 函数调用，并且对其取值后得到 JavaScript 对象



### 特例

```html
<!-- for 在 JSX 中应该被写作 htmlFor -->
<label htmlFor="namedInput">Name:</label>
<input id="namedInput" type="text" name="name"/>
```

### 无障碍辅助功能

[reactjs.org-无障碍](https://zh-hans.reactjs.org/docs/accessibility.html) 

> WCAG：[网络内容无障碍指南（Web Content Accessibility Guidelines，WCAG）](https://www.w3.org/WAI/intro/wcag) 为开发无障碍网站提供了指南。
>
> WAI-ARIA：[网络无障碍倡议 - 无障碍互联网应用（Web Accessibility Initiative - Accessible Rich Internet Applications）](https://www.w3.org/WAI/intro/aria) 文件包含了创建完全无障碍 JavaScript 部件所需要的技术。
>
> JSX 支持所有 `aria-*` HTML 属性。
>
> 语义化的 HTML 是无障碍辅助功能网络应用的基础。
