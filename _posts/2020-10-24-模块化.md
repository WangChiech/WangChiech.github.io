## 模块

把各个功能都单独的去写，然后再按一定的规则拼起来

> 模块化的好处
> 	1、避免命名冲突
> 	2、功能独立，按需加载
> 	3、提高复用性与维护性

1. 最原始的写法

   ```js
   function fn1(){
   	//....
   }
   function fn2(){
   	//....
   }
   ```

   > 污染全局变量

2. 对象的写法

   ```js
   var module1={
   	name:'kaivon',
   	m1:function(){
   		//...
   	},
   	m2:function(){
   		//...
   	}
   }
   ```

   > 结构清晰、层次分明
   >
   > 容易被任意修改

3. 立即执行函数

   ```js
   var module2=(function(){
   	var _name='kaivon'
   	var m1=function(){
   		//...
   	}
   	var m2=function(){
   		//...
   	}
   	
   	return {
   		m1:m1,
   		m2:m2
   	}
   })();
   console.log(module2._name)	//undefined
   ```

   > 利用闭包保护了内部属性
   >
   > 模块化的雏形

4. 放大模式（可以继承）

   ```js
   var module3=(function(mod){
   	mod.m3=function(){
   		//...
   	}
   	return mod;
   })();
   var newModule=module3(module2)
   ```

   > 扩展模块

5. 宽放大（允许输入一个空对象）

   ```js
   var module4=(function(mod){
   	mod.m3=function(){
   		//...
   	}
   	return mod;
   })(window.module2||{})
   ```

6. 传入其它的变量

   ```js
   var module5=(function(window,undefined,$){
   	mod.m3=function(){
   		//...
   	}
   	return mod;
   })(window,undefined,jQuery)
   ```

7. `CommonJS`

   1. 用来支持服务端的模块化，在服务端直接用`nodejs`就可以
   2. 客户端用的话需要编译，借助`Browserify`工具

   **暴露模块** 

   1. `moudle.exports=value`

      ```js
      //这就是一个模块，或者一个功能。在模块里其实就是对外面暴露出一个东西
      module.exports={
      	message:'这是模块一',
      	fn(){
      		console.log(this.message);
      	}
      }
      ```

   2. `exports.xxx=value`

      ```js
      //这个模块我要暴露出两条数据
      exports.message='这是模块三';
      exports.arr=[1,2,3];
      ```

   > 暴露的都是一个exports对象

   **引入模块** 

   1. `require(xxx)`

      ```js
      //引入第三方的模块的时候，一般都会放在最上面，放在自定义模块的上面
      const un=require('underscore');
      
      //这里要把三个模块都引入进来，再使用
      const module1=require('./module/module1.js'),
      	module3=require('./module/module3.js');
      	
      module1.fn(); // 这是模块一
      console.log(module3.message); // 这是模块三
      ```

   > 注意：
   > 	1、如果引入的是第三方的模块，参数为包的名字
   > 	2、如果引入的是本地的模块，参数为模块的文件路径

   **客户端使用`CommonJS`**

   `ECMAScript` 不支持，需要借助`Browserify`编译

   `Browserify`

   - 官网：http://browserify.org/
   - 安装：`npm install -g browserify`
   - 编译：`browserify js/src/app.js -o js/dist/index.js`
     注意：这里一定要弄对路径，要看当前命令工具里的目录在哪里

8. `AMD`规范

   专门用于客户端模块化的规范，模块的加载是异步形式

   **暴露模块（定义模块）** 

   1. 没有依赖的模块

      ```js
      define(function(){
      	return 模块;
      })
      ```

   2. 有依赖的模块

      ```js
      define(['module1','module2'],function(m1,m2){
      	return 模块;
      })
      ```

   **引入模块**

   ```js
   requirejs(['module1','module2'],function(m1,m2){
   	//...
   })
   ```

   **`Require.js`**

   - 官网：https://requirejs.org/

   - `require.js`的使用步骤

     - 在页面里通过script标签引入`require.js`

       ```html
       <script src="js/lib/require.js" data-main="js/app.js"><\/script>
       // data-main为主模块文件的路径（必需要指定）
       ```

     - 主模块文件里

       1. 基本配置

          ```js
          requirejs.config()
          ```

       2. 引入模块

          ```js
          requirejs(['模块名'],function(模块名){
          	...
          })
          ```

   - egg:

     ```html
     <!-- data-main	指定主模块的文件，这个一定要指定！！！-->
     <script src="js/lib/require.js" data-main="js/app.js"></script>
     ```

     ```js
     //app.js文件是用来引入其它模块的
     //基本配置
     requirejs.config({
     	//baseUrl:'js/',		//基础路径（指定所有模块的统一路径）
     	paths:{	//配置所需模块的路径
     		module1:'module/module1',
     		module2:'module/module2',
     		/*
     		 * 注意：
     		 * 	1、key的名字就代表了这个模块，再引入模块的地方直接用这个名字就可以了
     		 * 	2、value的值为模块的地址，不需要加后缀名
     		 */
     		jquery:'lib/jquery-1.11.0', // jquery做了AMD检测
     		underscore:'lib/underscore'
     	},
     	shim:{	//用来配置不兼容的模块（没有AMD检测）
     		'underscore':{
     			exports:'_',
     			//deps:['jquery','module1']
     		}
     		/*
     		 * 1、exports	对外暴露的内容，代表这个模块外部调用时候的名称
     		 * 2、deps		数组，表明该模块的依赖性
     		 */
     	}
     });
     
     //引入模块
     //requirejs(['module2'],module2=>module2.showMessage());
     requirejs(['module2','jquery','underscore'],(module2,$,_)=>{
     	module2.showMessage()
     	$('body').css('background','green')
     	
     	const result=_.map([1,2,3],num=>num*3)
     	console.log(result)
     })
     ```

     ```js
     //module1.js,没有依赖的模块
     define(()=>{
     	//这个函数直接返回的就是一个字符串
     	const logMessage=()=>`这是个没有依赖的模块`
     	
     	//暴露模块
     	return {logMessage}
     })
     ```

     ```js
     //module2.js,有依赖的模块
     //第一个参数里的名字，就是app.js里定义的，一定要对应上
     define(['module1'],module1=>{	//回调函数的参数要与前面数组里一一对应
     	const message='这是一个有依赖的模块'
     	const showMessage=()=>console.log(message,module1.logMessage())
     	
     	//暴露模块
     	return {showMessage}
     })
     ```

10. `ES6` 模块化

    **导出模块**

    ```js
    //导出模块:方法1
    export const m1Name='模块1';
    export const arr=[1,2,3];
    
    export function m1(){
    	console.log('这是模块1的m1');	
    };
    export function m2(){
    	console.log('这是模块1的m2');	
    };
    ```

    ```js
    //导出模块:方法2
    const m2Name='模块2';
    
    function m1(){
    	console.log('这是模块2的m1');
    }
    function m2(){
    	console.log('这是模块2的m2');
    }
    
    //export {m2Name,m1,m2};
    export {
    	m2Name,
    	m1 as fn1,
    	m2 as fn2,
    	//m2 as fn3,
    }
    ```

    ```js
    //导出模块:方法3
    export default {
    	name:'模块3',
    	fn(){
    		console.log('这是模块3');
    	}
    };
    ```

    **导入模块**

    ```js
    //引入模块
    import {m1Name,arr as num,m1,m2} from './module1';	//导入了第一个模块
    import {m2Name,fn1,fn2} from './module2';	//导入了第二个模块
    import * as module2 from './module2';	//整体加载
    import module3 from './module3'
    
    //引入系统模块
    import $ from 'jquery';
    $('body').css('background','green')
    
    //模块1的内容
    m1(),m2(),console.log(m1Name,num);
    
    //模块2的内容
    fn1(),fn2(),console.log(m2Name);
    
    //整体加载
    module2.fn1(),module2.fn2(),console.log(module2.m2Name);
    
    //模块3的内容
    module3.fn(),console.log(module3.name);
    ```
