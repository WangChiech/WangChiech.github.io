#### **简介** 

Koa是一个全新的web框架，致力于成为web应用和API开发领域中的一个更小、更富有表现力、更健壮的基石。

利用async函数丢弃回调函数，并增强错误处理。Koa没有任何预置的中间件，可快速而愉快的编写服务端应用程序。

>nodejs为什么有这么高的工作效率，最主要的特点就是异步非阻塞，async函数解决了的问题是，使用同步的方式书写异步的函数，通俗的将nodejs在执行代码的时候是从上到下去执行的，当执行到async函数的时候直接丢到队列里面去，直接执行后面函数，目的是不影响后面方法的执行达到异步的目的，有更高的执行效率和体验
>
>Koa非常的轻量，没有封装任何的接口和协议处理方面的工具或插件，需要使用另外的依赖包，使用npm的命令去进行安装，去扩展Koa这个框架的能力
>
>Koa最主要最核心的是去实现了协议（http）的处理

**核心概念** 

1. Koa Application (应用程序)
2. Context(上下文)：像一个管道一样，这个插件处理完之后交给另外一个插件
3. Request(请求)、Response(响应)

#### **基本使用** 

1. `npm init -y` 初始化项目，快速生成package.json文件

2. `npm install --save koa`  //  `npm install -S koa` 

3. 创建index.js文件

   ```js
   // index.js
   const Koa = require('koa')
   const app = new Koa()
   
   app.use(async (ctx) => {
       console.log(ctx) // {request:...,respond:...}
       ctx.body = 'hello word!'
   })
   
   app.listen(3000)
   ```

4. 打开终端执行`node index.js` 

5. 在浏览器地址栏输入`localhost:3000`

6. `npm install -S koa-router` 

   ```js
   // index.js
   const Koa = require('koa')
   const Router = require('koa-router')
   const app = new Koa()
   const router = new Router()
   
   router.get('./', ctx => {
       console.log(ctx)
       ctx.body = 'hello word'
   })
   
   router.get('./api', ctx => {
       console.log(ctx)
       ctx.body = 'hello api'
   })
   
   app.use(router.routes())
   	.use(router.allowedMethods())
   
   app.listen(3000)
   ```

7. 新建middleware.js

   ```js
   const Koa = require('koa')
   const app = new Koa()
   
   const middleware = function async(ctx, next) {
       console.log('this is a middleware')
       console.log(ctx.request.path)
       next()
   }
   const middleware1 = function async(ctx, next) {
       console.log('this is a middleware1')
       console.log(ctx.request.path)
       next()
       console.log('1 end')
   }
   const middleware2 = function async(ctx, next) {
       console.log('this is a middleware2')
       console.log(ctx.request.path)
       next()
       console.log('2 end')
   }
   // 中间件顺序执行，遇到next()直接跳到下个中间件执行，没有next()koa默认本次请求终止，返回给用户数据
   app.use(middleware1)
   app.use(middleware2)
   app.use(middleware)
   
   app.listen(3000) 
   // 最终打印顺内this is a middleware1=》this is a middleware2=》this is a middleware=》2 end=》1 end
   ```

8. 打开终端执行`node middleware.js` 

10. `npm install -S koa-body @koa/cors` 

    `npm install -S koa-json` 

    `npm install -S koa-compose` 整合koa中间件
    
    `npm install -S koa-compress` 压缩koa中间件
    
    ```js
    // index.js
    // 中间件：koa-router | koa-body | @koa/cors | koa-json
    const Koa = require('koa')
    const Router = require('koa-router')
    const cors = require('@koa/cors')
    const koaBody = require('koa-body')
    const koaJson = require('koa-json') // 将返回和请求的数据格式化
    const compose = require('koa-compose') // 整合集成koa中间件
    const compress from 'koa-compress' // 压缩中间件
    
    const isDevMode = process.env.NODE_ENV === 'production' ? false : true
    
    const app = new Koa()
    const router = new Router()
    
    router.prefix('/api') // 所有请求必须都带api
    
    router.get('./', ctx => {
        const params = ctx.request.query // 获取params
        console.log(ctx)
        ctx.body = 'hello word'
    })
    router.get('./api', ctx => {
        console.log(ctx)
        ctx.body = 'hello api'
    })
    router.get('./async', async (ctx) => {
        let result = await new Promise((resolve) => {
            setTimeout(() => {
                resolve('hello word 2s later')
            }, 2000)
        })
        ctx.body = result
    })
    router.post('./post', async (ctx) => {
        let { body } = ctx.request
        console.log(body)
        ctx.body = {
            ...body
        }
    })
    
    // app.use(koaBody())
    // app.use(cors())
    // app.use(koaJson({ pretty: false, param: 'pretty' }))
    
    const middleware = compose([
        koaBody(),
        cors(),
        koaJson({ pretty: false, param: 'pretty' }),
        router.routes(),
        router.allowedMethods()
    ])
    
    if (!isDevMode) {
        app.use(compress())
    }
    
    app.use(middleware)
    
    app.use(router.routes())
    	.use(router.allowedMethods())
    
    app.listen(3000)
    ```

#### Koa路由进阶配置 

如果把全部路由请求全部写到index.js中，随着请求的增多，功能的复杂，这样做是非常不可取的，index.js会变得异常的冗长，合理的工作目录非常的重要。

1. 按照功能模块进行区分
   - 配置放到config文件里面
   - api放到controler或api文件里面
   - 路由放到routes文件里面

2. 路由的压缩：koa-combine-routers

   若使用一个中间件就使用一个app.use()，会导致index.js会非常的冗长，如果我们需要只引入一次，就需要一个路由压缩的过程

3. 静态资源的处理：koa-static

   不可能为了一个后台的资源写一个后台的服务，最简单、最直接、最好的方式就是使用中间件，直接引入static中间件指定一个static目录，就可以对这些资源进行访问了，达到的效果就像是一个网盘一样，只不过这个静态资源的网盘没有任何的密码，任何人都可以去访问，可以去放一些公共的图片、公共的资源、一些不会很大的东西

**Docker容器技术的特点：**

1. 文件系统隔离：每个进程容器运行在完全独立的根文件系统里。

2. 资源隔离：可以使用cgroup为每个进程容器分配不同的系统资源，例如CPU和内存。

3. 网络隔离：每个进程容器运行在自己的网络命名空间里，拥有自己的虚拟接口和IP地址。

4. 写时复制：采用写时复制方式创建根文件系统，这让部署变得极其快捷，并且节省内存和硬盘空间。

5. 日志记录：Docker将会收集和记录每个进程容器的标准流（stdout/stderr/stdin），用于实时检索或批量检索。

6. 变更管理：容器文件系统的变更可以提交到新的映像中，并可重复使用以创建更多的容器。无需使用模板或手动配置。

7. 交互式Shell：Docker可以分配一个虚拟终端并关联到任何容器的标准输入上，例如运行一个一次性交互shell。

### NoSQL数据库设计

> 1. 易扩展，高性能、高可用
>
>    易扩展，数据存储的模型非常的灵活，数据结构不像关系型数据库是一个固定的，NoSQL数据库他的数据结构是不固定的，非常容易扩展，数据与数据之间没有耦合，文档存储的方式非常容易部署以及使用；高性能，非常方便的用来存储一些消息类型的数据，因为他的数据结构不固定，所以有的时候拿MongoDB做一些爬虫、分析也是非常方便的，在这个网站爬下来的数据和在那个网站爬下来的数据，数据结构都不太一样，可以把它放到MongoDB 里面去，再去做一些数据分析；高可用是指在不影响一些性能的情况下，就可以很方便的去实现数据库的扩展，提高它的性能，比如现在的一些Hbase的模型也能实现一些高可用的
>
> 2. 较容易映射复杂数据（key-value)
>
> 3. 无事务特性要求（ACID特性，MongoDB4以前不支持）
>
>    A-原则性：不可分割的特性
>
>    C-一致性
>
>    I-隔离性（独立性）
>
>    D-持久性
>
>    主要应用场景：秒杀、购物（以便对数据、对用户整个操作过程有个追踪

**关系型数据库** ，是指采用了关系模型来组织数据的数据库

**NoSQL** ， 是对不同于传统的关系数据库的数据管理系统的统称

#### 常见场景及设计方法（内嵌、父/子引用、反范式）

> 1. 优先考虑内嵌，如果单独访问，则不适合
> 2. 数组不应该无限制增长
> 3. 考虑读写比，考虑反范式，考虑应用场景

**内嵌** 

指存在关联关系的文档，放在同一文档中，以数组的形式存放。（下面roles、address都是内嵌）

```json
{
    "username": "brian",
    "gender": 0,
    "roles": ["admin", "user"],
    "address": [
        "address1",
        "address2"
    ]
}
```

- 减少了关联查询
- 适合于单类需要描述的属性
- 不经常变化的属性（扩展、嵌套关联）

**父引用** 

指存在一对多的情况中，放在同一文档中，以数组的形式存放

```json
{
    "tid": "postID",
    "title": "文章标题",
    "catalog": "index",
    "created": "123",
    "comments": [
        "commentID1",
        "commentID2"
    ]
}
```

**子引用** 

指存放在一对多的情况中，由于数据库文档存放限制，这时候进行反向引用

```json
{
    "cid": "commentID",
    "tid": "postID",
    "content": "这是回复内容！！",
    "isRead": 0,
    "isBest": 0,
    "status": 1
}
```

> 父子引用设计
>
> 1. 引用数据内容是否非常多
> 2. 引用数据量是否非常庞大，而且在增加
> 3. 数据是否需要单独访问

**反范式** 

范式是指按照既定的用法，范式就是一种公认的模型或模式。`反范式-不走寻常路` 

```json
{
    "tid": "postID",
    "title": "文章标题！！",
    "created": 123,
    "users": [
        {
            "uid": "用户ID",
            "name": "用户昵称",
            "isVip": "1"
        }
    ]
}
```

> 反范式设计
>
> 1. 是否有提升性能的区间
> 2. 数据量的变化是否非常庞大，庞大到更新会异常低效
> 3. 先考虑读写比，才考虑反范式
