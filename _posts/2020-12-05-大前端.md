#### **简介** 

Koa是一个全新的web框架，致力于成为web应用和API开发领域中的一个更小、更富有表现力、更健壮的基石。

利用async函数丢弃回调函数，并增强错误处理。Koa没有任何预置的中间件，可快速而愉快的编写服务端应用程序。

>nodejs为什么有这么高的工作效率，最主要的特点就是异步非阻塞，async函数解决了的问题是，使用同步的方式书写异步的函数，通俗的将nodejs在执行代码的时候是从上到下去执行的，当执行到async函数的时候直接丢到队列里面去，直接执行后面函数，目的是不影响后面方法的执行达到异步的目的，有更高的执行效率和体验
>
>Koa非常的轻量，没有封装任何的接口和协议处理方面的工具或插件，需要使用另外的依赖包，使用npm的命令去进行安装，去扩展Koa这个框架的能力
>
>Koa最主要最核心的是去实现了协议（http）的处理

**核心概念** 

1. Koa Application (应用程序)
2. Context(上下文)：像一个管道一样，这个插件处理完之后交给另外一个插件
3. Request(请求)、Response(响应)

#### **基本使用** 

1. `npm init -y` 初始化项目，快速生成package.json文件

3. 创建index.js文件

   ```js
   const Koa = require('koa')
   const app = new Koa()
   
   app.use(async (ctx) => {
       console.log(ctx) // {request:...,respond:...}
       ctx.body = 'hello word!'
   })
   
   app.listen(3000)
   ```

4. 打开终端执行`node index.js` 

5. 在浏览器地址栏输入`localhost:3000`

6. `npm install -S koa-router` 

   ```js
   // index.js
   const Koa = require('koa')
   const Router = require('koa-router')
   const app = new Koa()
   const router = new Router()
   
   router.get('./', ctx => {
       console.log(ctx)
       ctx.body = 'hello word'
   })
   
   router.get('./api', ctx => {
       console.log(ctx)
       ctx.body = 'hello api'
   })
   
   app.use(router.routes())
   	.use(router.allowedMethods())
   
   app.listen(3000)
   ```

7. 新建middleware.js

   ```js
   const Koa = require('koa')
   const app = new Koa()
   
   const middleware = function async(ctx, next) {
       console.log('this is a middleware')
       console.log(ctx.request.path)
       next()
   }
   const middleware1 = function async(ctx, next) {
       console.log('this is a middleware1')
       console.log(ctx.request.path)
       next()
       console.log('1 end')
   }
   const middleware2 = function async(ctx, next) {
       console.log('this is a middleware2')
       console.log(ctx.request.path)
       next()
       console.log('2 end')
   }
   // 中间件顺序执行，遇到next()直接跳到下个中间件执行，没有next()koa默认本次请求终止，返回给用户数据
   app.use(middleware1)
   app.use(middleware2)
   app.use(middleware)
   
   app.listen(3000) 
   // 最终打印顺内this is a middleware1=》this is a middleware2=》this is a middleware=》2 end=》1 end
   ```

8. 打开终端执行`node middleware.js` 

10. `npm install -S koa-body @koa/cors` 

    `npm install -S koa-json` 

    ```js
    // index.js
    // 中间件：koa-router | koa-body | @koa/cors | koa-json
    const Koa = require('koa')
    const Router = require('koa-router')
    const cors = require('@koa/cors')
    const koaBody = require('koa-body')
    const koaJson = require('koa-json') // 将返回和请求的数据格式化
    
    const app = new Koa()
    const router = new Router()
    
    router.prefix('/api') // 所有请求必须都带api
    
    router.get('./', ctx => {
        const params = ctx.request.query // 获取params
        console.log(ctx)
        ctx.body = 'hello word'
    })
    router.get('./api', ctx => {
        console.log(ctx)
        ctx.body = 'hello api'
    })
    router.get('./async', async (ctx) => {
        let result = await new Promise((resolve) => {
            setTimeout(() => {
                resolve('hello word 2s later')
            }, 2000)
        })
        ctx.body = result
    })
    router.post('./post', async (ctx) => {
        let { body } = ctx.request
        console.log(body)
        ctx.body = {
            ...body
        }
    })
    
    app.use(koaBody())
    app.use(cors())
    app.use(koaJson({ pretty: false, param: 'pretty' }))
    
    
    app.use(router.routes())
    	.use(router.allowedMethods())
    
    app.listen(3000)
    ```

#### **Koa路由进阶配置 **

如果把全部路由请求全部写到index.js中，随着请求的增多，功能的复杂，这样做是非常不可取的，index.js会变得异常的冗长，合理的工作目录非常的重要。

1. 按照功能模块进行区分
   - 配置放到config文件里面
   - api放到controler或api文件里面
   - 路由放到routes文件里面

2. 路由的压缩：koa-combine-routers

   若使用一个中间件就使用一个app.use()，会导致index.js会非常的冗长，如果我们需要只引入一次，就需要一个路由压缩的过程

3. 静态资源的处理：koa-static

   不可能为了一个后台的资源写一个后台的服务，最简单、最直接、最好的方式就是使用中间件，直接引入static中间件指定一个static目录，就可以对这些资源进行访问了，达到的效果就像是一个网盘一样，只不过这个静态资源的网盘没有任何的密码，任何人都可以去访问，可以去放一些公共的图片、公共的资源、一些不会很大的东西
