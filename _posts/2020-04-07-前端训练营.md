## 第四天 0416

### 编程语言通识

- 非形式语言
  - 中文、英文

- 形式语言（乔姆斯基谱系）
  - 0型 无限制文法 
    - ?::=?
  - 1型 上下文相关文法
    - ?<A>::=?
  - 2型 上下文无关文法
    - <A>::=?
  - 3型 正则文法
    - <A>::=<A>?<A>×

> 现代语言一般会采取一个比较折中的办法，会把所有的文法分成词法和语法两个部分，词法是用正则做一遍粗略的处理，把语言变成单个的词，再把这些词作为一个输入流去做语法分析，所以今天作为主流的语言都会分为词法和语法两个部分

**产生式(BNF)** 

- 用尖括号括起来的名称来表示语法结构名
- 语法结构分成基础结构和需要用其他语法结构定义的复合结构
  - 基础结构称终结符
  - 复合结构称非终结符

- 引号和中间的字符表示终结符
- 可以有括号
- *表示重复多次
- |表示或
- +表示至少一次

```js
// 定义0-9写法
<number> = "0" | "1" | "2"| ……|"9"
// 定义十进制整数写法
<DecimalNumber> = "0" | (("1" | "2"| ……|"9")<number>*)
// 定义加法
<Expresion> = <DecimalNumber> | <Expresion> "+" <DecimalNumber>          
```

四则运算：

- 1 + 2 * 3

终结符：

- Number
- +、-、* 、/

非终结符：

- MultiplicativeExpression
- AddtiveExpression

>  JavaScript和接触到的绝大多数计算机语言都以2型为主，凡是上下文相关文法在上下文处理上都是让语法分析器回传一个标志给词法分析器来做的

**现代语言的特例**

- C++中，*可能表示乘号或者指针，具体是指哪个，取决于星号前面的标识符是否被声明为类型

- VB中，< 可能是小于号，也可能是XML直接量的开始，取决于当前位置是否可以接受XML直接量

- Python中，行首的tab符合空格会根据上一行的行首空白以一定规则被处理成虚拟终结符indent或者dedent
- JavaScript中，/可能是除号，也可能是正则表达式的开头，处理方式类似于VB，字符串模板中也有需要特殊处理}，还有自动插入分号规则

**图灵完备性**

和图灵机等效的都具有图灵完备性，凡是可计算的都能计算出来，叫图灵机

- 命令式——图灵机
  - goto
  - if和while

- 声明式——lambda
  - 递归

**动态与静态**

- 动态：
  - 在用户的设备/在线服务器上
  - 产品实际运行时
  - Runtime

- 静态：
  - 在程序员的设备上
  - 产品开发时
  - Compiletime

**类型系统**

- 动态类型系统与静态类型系统
- 强类型与弱类型
  - String + Number
  - String == Booleabn

- 复合类型
  - 结构体
  - 函数签名

- 子类型
  - 逆变/协变
