## 1-1 准备工作

### 认识Flow

Flow 是facebook 出品的 JavaScript 静态类型检查工具，Vue.js 的源码利用了Flow做了静态检查。

JavaScript 是动态类型语言，JavaScript 目前通常会写ES6，用babel编译，在编译期没有明确的语法错误是不会报错的，但实际运行阶段会出现各种奇怪的bug。

类型检查是当前动态类型语言的发展趋势。所谓动态类型语言就是在编译期尽早的发现（由类型错误引起的）bug，由不能影响代码运行（不需要运行时童泰检查类型），是编译JavaScript具有和Java等强类型语言相近的体验。

项目越复杂越需要通过工具的手段来保证项目的维护性和增强代码的可读性。Vue.js 在做2.0重构的时候，在ES5015的基础上，除了ESLint保证代码风格外，也引入了Flow做静态类型检查。之所以选择Flow，主要是因为Babel和ESLint都有相应的Flow插件以支持语法，可以完全沿用现有的配置，非常小的成本改动就可以拥有静态类型检查的能力。

### Flow 的工作方式

通常类型检查分成2种方式：

- 类型推断：通过变量的使用上下文来推断出变量类型，然后根据这些判断来检查类型。
- 类型注释：实现注释好我们期待的类型，Flow 会基于这些注释来判断。

#### 类型判断

它不需要任何代码修改即可进行检查，最小的工作量

```js
// 安装
npm install -g flow-bin
// 创建.flowconfig 文件(描述不想对哪些文件做检查，想对哪些文件做检查)
flow init
```
