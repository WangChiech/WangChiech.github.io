## React源码解析

学习源码更好的理解写的代码是如何运行的，遇到bug的时候可以从底层去思考如何去解决问题。并且可以在开始项目之前想好更好的架构方式。

react源码的很多设计细节都是非常高明的，学习源码有助于提升整体的编码能力，养成良好的编码思维。

react是一个非常纯粹的 `UI = fn(x)` 的框架，通过 `state` 映射 `UI` 的方式来屏蔽了 `DOM` 操作，现在业界框架大多如此，但是说 react 纯粹是因为他的 `API` 设计非常的纯粹，核心 `API` 就是 `setState` ，其余所有的内容都围绕着组件化来设计，没有 `directive` 双向绑定和一些其他的 `API` ，你要改变 `UI` 只能通过 `setState` 来改变对应的状态，这就给了react 无比纯粹的开发体验，一切基于组件。同时 react 又是一个思想超前的框架，16版本完全重写了核心代码，但是对使用者是毫无感知的，不需要调整兼容。更优秀的是16版本引入了 `Fiber` 的概念，从根本上解决了 `JS` 单线程运行，如果计算量太大的话导致动画卡帧和交互卡顿的问题。
